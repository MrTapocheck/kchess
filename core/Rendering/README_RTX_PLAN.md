# План реализации RTX / 3D рендеринга для kchess

## Цель
Заменить 2D спрайты на полноценное 3D представление шахматной доски с трассировкой лучей (Ray Tracing) для реалистичных отражений, теней и преломлений.

## Выбор технологии
Рассматриваются два пути:
1. **Vulkan + Ray Tracing Extensions**: Максимальная производительность, доступ к RT-ядрам NVIDIA/AMD. Сложная инициализация, много бойлерплейта.
2. **OpenGL 4.6 + Compute Shaders**: Более простой вход, возможность написать софтверный рейкастер внутри шейдера. Достаточно для статичной сцены шахмат.

## Этапы реализации

### Этап 1: Инициализация контекста
- [ ] Интеграция OpenGL/Vulkan контекста в окно Avalonia (через `NativeControlHost` или кастомный рендерер).
- [ ] Отображение простого треугольника ("Hello Triangle").
- [ ] Настройка цикла отрисовки (Swapchain, Present).

### Этап 2: Базовая 3D сцена
- [ ] Загрузка 3D моделей фигур (формат .glb или .obj).
- [ ] Камера и матрицы проекции (View/Projection).
- [ ] Базовое освещение (Phong/Blinn-Phong).

### Этап 3: Внедрение Ray Tracing (RTX)
- [ ] Написание Ray Generation Shader (пуск лучей из камеры).
- [ ] Реализация пересечения Луч-Треугольник (алгоритм Мёллера–Тромбора).
- [ ] Построение структур ускорения (BVH) для быстрого поиска пересечений.
- [ ] Реализация материалов (Diffuse, Metal, Glass) через шейдеры.
- [ ] Добавление рекурсивных отражений и теней.

### Этап 4: Интеграция с логикой
- [ ] Синхронизация позиций фигур из `ChessEngine` с буферами вершин GPU.
- [ ] Анимация ходов (интерполяция позиций).

## Ресурсы
- Модели фигур: *ожидается загрузка в папку Assets/Models3D*
- Библиотеки: `Silk.NET.Vulkan`, `Silk.NET.OpenGL`, `SharpGLTF`.
